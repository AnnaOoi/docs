The TechDocs platform is designed with a straightforward layout to help you quickly access documents. Navigating to techdocs.grab.com will bring you to the TechDocs home page.

## Content types

The top navigation menu provides access to different content types:

- **Product docs**: User guides, developer guides, external runbooks, and resources for platforms, tools, libraries, and SDKs.
- **Service docs**: User-facing documentation for microservices.
- **Learning paths**: Step-by-step tutorials for platforms, tools, or specific areas.
- **Best practices**: Guidelines and mental models.
- **Blogs**: Internal blogs accessible only to Grabbers.

## Three-panel layout structure

### Left navigation panel

- **Product listing**: Displays all available products in alphabetical order for easy browsing.
- **Hierarchical structure**: Provides a tree-like view of documentation, allowing users to drill down from products → documentation types → specific pages.
- **Filter feature**: Located at the top of the panel, enables keyword search across page titles for quick access to specific documentation.
- **Persistent navigation**: Remains visible while browsing to maintain context and enable quick switching between products.

### Main content area (center)

On the home page:

- **Explore domains** section: Displays documentation grouped under different categories.
- **Category overview**: Each category shows a quick summary and lists related products with brief descriptions.
- **Quick access**: Allows users to dive directly into relevant documentation without navigating through multiple layers.

On documentation pages:

- **Content display**: Shows the actual documentation content, articles, guides, or tutorials.
- **Structured content**: Follows markdown formatting with proper headings, sections, and navigation elements.
- **Breadcrumb navigation**: Shows current location and provides easy navigation back to parent sections.
- **Last updated**: Information showing when the page was last modified.

### Right sidebar panel

Metadata and actions:

- **Edit this page**: Quick access button (top right) for making edits via GitLab Web IDE or local setup.
- **Give feedback**: Feature for reporting issues, suggestions, or submitting feedback via GitLab issues.
- **Owner information**: Displays who owns/maintains the specific documentation.

Table of contents:

- **Page navigation**: Displays an outline of the current page's sections and headings.
- **Quick jumping**: Allows users to jump directly to specific sections within long documents.
