This page provides a high-level introduction to TechDocs, including what it is, who it's for, and how it can help you achieve your documentation goals.
Whether you're exploring TechDocs for the first time or looking to understand its core functionality, this overview will give you a clear picture of its value and purpose.

## What is TechDocs?

TechDocs is a comprehensive technical documentation platform that centralizes product docs, service docs, best practices, learning paths, and internal blogs.

Built to solve the fragmentation of technical documentation across multiple platforms, TechDocs provides a unified home for all your team's documentation needs. It combines the power of Hugo static site generation with GitLab-based content management, making it easy for teams to create, maintain, and discover technical documentation.

TechDocs serves as the single source of truth for technical knowledge at Grab, ensuring that developers, product managers, and other stakeholders can easily find the information they need to build and maintain products successfully.

## Audience

TechDocs is designed for:

- **Product teams**: Product managers and developers who want to publish user guides, API documentation, and technical specifications.
- **Engineering teams**: Software engineers documenting services, APIs, architectures, and development processes.
- **Platform teams**: Infrastructure and platform engineers who need to document shared services, tools, and development standards.
- **Content contributors**: Team members who need to update existing documentation or contribute new content.

## Purpose and value

TechDocs was created to solve documentation fragmentation and help teams create, maintain, and discover technical knowledge more effectively.

**Key benefits**:

- **Centralized knowledge**: All technical documentation in one searchable platform, eliminating the need to search across multiple tools.
- **Consistent experience**: Standardized templates and guidelines ensure users have a familiar experience across all product documentation.
- **Easy content creation**: Git-based workflows and markdown support make it simple for technical teams to contribute documentation.
- **Discoverability**: Powerful search and categorization features help users find relevant information quickly.

**Problems it solves**:

- Scattered documentation across multiple platforms and tools.
- Inconsistent formatting and structure across different team's documentation.
- Difficulty finding relevant technical information when needed.
- Lack of standardized processes for creating and maintaining documentation.
- Time wasted recreating similar documentation structures.

## Key features

- **Hugo-powered static site generation**: Fast, reliable, and scalable documentation websites with excellent performance.
- **GitLab integration**: Seamless content management through GitLab's Web IDE or local development environments.
- **Markdown support**: Write documentation using familiar markdown syntax with extended features for technical content.
- **Template system**: Pre-built templates and guidelines for consistent documentation structure across teams.
- **Quality assurance tools**: Built-in linting and validation to ensure documentation meets quality standards.
- **Advanced search**: Powerful search capabilities to help users find information quickly across all documentation.

## Use cases

Here are common ways teams use TechDocs:

### API and service documentation

Development teams document their APIs, microservices, and technical architectures, making it easy for other teams to understand and integrate with their services.

**Example**: A payments team documents their API endpoints, authentication methods, and integration examples for other product teams to use.

### Product user guides

Product teams create comprehensive user guides and feature documentation that help both internal stakeholders and external users understand product functionality.

**Example**: A mobile app team documents new features, user flows, and troubleshooting guides for customer support and product marketing teams.

### Best practices and learning paths

Teams share knowledge, best practices, and learning resources to help other teams avoid common pitfalls and accelerate development.

**Example**: The platform team creates guides on deployment processes, monitoring setup, and coding standards for all engineering teams to follow.

## How it works

TechDocs follows a straightforward content creation and publication workflow:

1. **Content creation**: Write documentation using markdown files in GitLab repositories.
2. **Review process**: Use GitLab's merge request workflow for peer review and approval.
3. **Automatic publishing**: Hugo automatically generates and deploys your documentation website.
4. **Discovery and search**: Users can find and access documentation through the TechDocs portal.
